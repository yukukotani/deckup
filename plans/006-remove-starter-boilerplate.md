## Why this matters

Two workspace packages are untouched scaffolding from the initial Vite+
template ("Initial commit from Vite+", commit 974ba2c): `apps/website` is a
stock Vite starter (counter button, Vite/TypeScript logos) and
`packages/utils` is a tsdown library starter whose only export returns
`"Hello, tsdown!"` and whose package.json still says
`"author": "Author Name <author.name@mail.com>"`. Nothing imports either
one. They add install time, `vp run -r` build/test time, lockfile surface,
and contributor confusion (the audit initially mistook `website` for a
planned docs site). Removing them makes `vp run ready` faster and the
workspace honest.

Note for the maintainer reviewing this plan: if a docs/marketing site IS
planned, keep the `apps/website` _name_ reserved by deleting the starter
content anyway — a real site would be regenerated from a real template, not
grown out of the counter demo.

## Steps

### Step 1: Re-verify nothing references the two packages

**Verify**:
`rg -n "website|@slida/utils" --glob '!node_modules' --glob '!pnpm-lock.yaml' --glob '!plans' --glob '!.rpiv' package.json apps example packages tools 2>/dev/null`
→ matches only inside `apps/website/` itself (its own package.json/README),
nothing elsewhere. If any OTHER file matches, STOP.

### Step 2: Delete the directories

```bash
git rm -r apps/website packages/utils
```

### Step 3: Refresh the lockfile

Run `vp install`.

**Verify**: exit 0; `git status` shows `pnpm-lock.yaml` modified and the two
directories deleted — nothing else.

### Step 4: Full verification

**Verify**: `vp run ready` → exit 0 (check + all tests + all builds across
the remaining workspace).
